Practice 3

Create the following Test ("..\js\src"):

* Automate 5 Smoke test

1. Verify all Project are getting

2. Verify all items are getting

3. Verify the user information is getting

4. Verify all Filter are retrieve

5. Verify token is returned with basic authentication

* Automate 5 Sanity test

1. Verify information (email& fullname&Id) about user is getting correctly

2. Verify Priority field to all items is returned correctly

3. Verify the content and number of items from Projects are listed properly

4. Verify ULR of icon about projects is getting correctly

5. Verify when a project is created its content and number of items have the correct value

* Automate 10 workflows(include negative tests)

1. Verify a new Project can be created as expected

2. Verify a new Item can be created as expected

3. Verify if the done items to specific project are displayed properly

4. Verify a item can be checked (updated)

5. Verify name to specific project can be updated

6. Verify an error is displayed if the Name to project is empty.

7. Verify an error is displayed trying to created a user without password

8. Verify an item can be deleted

9. Verify URL to icon about projects have format http://

10. Verify an error is displayed trying to create an item without name

Notes:

For workflows, please consider:

• Automating also the pre-requisite steps

• Do not leave buggy data in the app. For instance, If the test needs 5 projects created, create them and once test is done, delete them

• Apply jasmine-node matchers
